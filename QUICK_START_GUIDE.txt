🔧 TECHNICAL DETAILS:
═══════════════════════════════════════════════════════════════════════════════

Data Files (Created Automatically at Runtime):
  data/users.json              - Registered user accounts
  data/followup_items.json     - All followup items
  data/contacts.json           - External contacts (NEW)
  data/email_preferences.json  - User email preferences (NEW)

Email Routing Logic:
  1. Check item settings:
     - send_to_responsible? → Add responsible_email to recipients
     - cc_owner? → Add owner_email to recipients
     - additional_recipients? → Add all to recipients
  
  2. Check user preferences:
     - Does user have emails enabled?
     - Does user want this email type?
  
  3. Send email to final recipient list

Backward Compatibility:
  - Existing items without email settings work as before
  - Old data structure fully supported
  - No data migration needed
